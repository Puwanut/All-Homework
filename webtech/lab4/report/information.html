ABOUT Node.js


Node.js คือ JavaScript runtime สร้างบน Chrome’s V8 JavaScript engine โดย Node.js ใช้ event-driven, non-blocking I/O model ทำให้มันเบาเครื่องและมีประสิทธิภาพสูง

NODE.JS = Runtime Environment + JavaScript Library

เมื่อนำ Runtime Environment มาบวกกับ JavaScript Library ทำให้เราสามารถนำมาประยุกต์สร้าง Application ต่างๆ ได้ไม่ว่าจะเป็นระบบ Streaming, Online Chat, Web, Web Service, IOT และอื่นๆ

Node.js มี Library ที่เป็นส่วนเสริมจำนวนมาก แยกเป็น JavaScript Module ต่างๆ สำหรับทำงานเฉพาะ ในแต่ละเรื่องสามารถติดตั้งเพิ่มเติมได้ง่ายผ่าน npm (Node Package Manager)

Npm (Node Package Manager) เป็นตัวจัดการ Package ของ JavaScript คุณสามารถหา Package ที่เหมาะสมกับงานแต่ละอย่างได้ที่นี้ https://www.npmjs.com และติดตั้งผ่านคำสั่งบน command line ได้เลย


ทำไมถึงต้องใช้ Node.js ?
Node.js ใช้การเขียนโปรแกรมแบบ Asynchronous!

งานทั่วไป (Common task) สำหรับ Web Server สามารถเปิดไฟล์บน Server และส่งคืนเนื้อหาไปยัง Client ได้
นี่คือวิธีที่ PHP หรือ ASP จัดการคำขอไฟล์

  1. ส่งงานไปยังระบบไฟล์ของคอมพิวเตอร์
  2. รอในขณะที่ระบบไฟล์เปิดขึ้นและอ่านไฟล์
  3. ส่งคืนเนื้อหาไปยังไคลเอนต์
  4. พร้อมจัดการตามคำขอต่อไป

นี่คือวิธีที่ Node.js จัดการคำขอไฟล์:

  1. ส่งงานไปยังระบบไฟล์ของคอมพิวเตอร์
  2. พร้อมจัดการตามคำขอต่อไป
  3. เมื่อระบบไฟล์เปิดและอ่านไฟล์เซิร์ฟเวอร์จะส่งคืนเนื้อหาไปยังไคลเอ็นต์

Node.js ช่วยลดการรอและดำเนินการตามคำขอถัดไป
Node.js รันแบบ single-thread ไม่ปิดกั้นการเขียนโปรแกรมแบบ Asynchronous ซึ่งเป็นการบริหารหน่วยความจำที่มีประสิทธิภาพมาก


Node.js สามารถทำอะไรได้บ้าง ?
  • Node.js สามารถสร้างเนื้อหาเพจแบบ Dynamic
  • Node.js สามารถสร้าง, เปิด, อ่าน, เขียน, ลบ, และปิดไฟล์บน Server
  • Node.js สามารถรวบรวมข้อมูลแบบฟอร์ม
  • Node.js สามารถเพิ่ม, ลบ, แก้ไขข้อมูลในฐานข้อมูลได้

Node.js เอาไปต่อยอดอะไรได้บ้าง ?
  • สร้าง Mobile App แบบข้ามระบบ iOS และ Android ได้ โดยใช้เครื่องมือที่ชื่อว่า PhoneGap/Cordova หรือ Ionic Framework
  • สร้างฝั่ง Web Server โดยใช้ Framework อย่าง MEAN Stack หรือ Meteor
  • สร้างระบบฝั่ง IoT โดยใช้ Particle ที่ชื่อเดิมคือ Spark JS, Cylon JS, หรือ Johny Five

ข้อดีของ Node.js
  • มีการทำงานแบบ Event-Driven และ Asynchronous (ไม่ต้องรอการทำงานคำสั่งอื่นที่ใช้เวลานานให้เสร็จก่อน สามารถไปทำงานคำสั่งถัดไปก่อนได้เลย)
  • รองรับ Concurrent ได้จำนวนมาก
  • เหมาะกับการทำ Web แบบ Real time
  • ประหยัดทรัพยากรในการทำงาน
  • มีการประมวลผลที่รวดเร็ว
  • เรียนรู้ง่าย เนื่องจาก Syntax ที่ใช้คือ JavaScript
